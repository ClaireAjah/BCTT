Project: HealthSync (Smart Health Platform)

Penjelasan Aplikasi:
HealthSync adalah platform manajemen kesehatan pintar yang dirancang untuk menjembatani komunikasi medis antara Pasien dan Dokter. Dibangun dengan fokus pada kecepatan, keamanan data, dan kemudahan aksesibilitas. Aplikasi ini mengandalkan Supabase (PostgreSQL) sebagai basis datanya yang memungkinkan pengambilan data secara real-time.

Fitur-fitur Utama:
1. Registrasi & Autentikasi JWT: Keamanan sistem login berbasis token untuk membedakan hak akses Pasien dan Dokter/Admin.
2. Dashboard Pasien Terintegrasi: 
   - Pencatatan vital sign (Tekanan Darah, Detak Jantung, Gula Darah, Berat Badan).
   - Visualisasi grafik pemantauan kesehatan dinamis (menggunakan Chart.js).
   - Pembuatan dan pengelolaan Janji Temu dengan dokter spesialis.
   - Akses langsung ke Rekam Medis dan Resep dari dokter.
3. Portal Manajemen Dokter:
   - Persetujuan, penyelesaian, dan pembatalan janji temu.
   - Fitur Input Rekam Medis pasien yang aman dan tersentralisasi.
   - Telemonitoring (Fitur untuk mencari pasien dengan ID dan memantau grafik riwayat kesehatannya dari jarak jauh).
   - Pembuatan pengumuman kesehatan dan kampanye vaksinasi.

Teknologi yang Digunakan:
- Frontend: HTML5, Vanilla CSS3 (Custom Glassmorphism Medical Dark Theme), Vanilla JavaScript (No Framework).
- Backend: Node.js, Express.js.
- Database: Supabase (PostgreSQL) & @supabase/supabase-js.
- Keamanan: JSON Web Token (JWT) dengan cookie HTTP-only.
- Visualisasi: Chart.js.

Note:
Kode kami dibuat dengan tingkat detail yang tinggi, responsif, logika yang tersambung dari database ke UI, dan 100% bebas dari komentar sesuai dengan instruksi yang diberikan.

DOKUMENTASI API (HealthSync)

1. Autentikasi (Auth)
   - POST /api/auth/register : Mendaftarkan pengguna baru (Pasien).
   - POST /api/auth/login    : Autentikasi pengguna (JWT) & set HTTP-only cookie.
   - POST /api/auth/logout   : Menghapus cookie sesi pengguna.
   - GET  /api/me            : Mendapatkan profil pengguna yang sedang login.

2. Pengguna (Users)
   - GET  /api/doctors       : Mendapatkan daftar semua dokter yang terdaftar.

3. Janji Temu (Appointments)
   - POST /api/appointments      : Membuat janji temu baru (Pasien -> Dokter).
   - GET  /api/appointments      : Melihat jadwal janji temu (Berdasarkan Role).
   - PUT  /api/appointments/:id/status : Mengubah status janji (Confirmed/Completed/Cancelled).

4. Rekam Medis (Medical Records)
   - POST /api/records : Input data rekam medis dan resep (Khusus Dokter).
   - GET  /api/records : Mendapatkan daftar rekam medis pasien terkait.

5. Vital Signs (Pemantauan Pasien)
   - POST /api/vitals : Mengirim data vital sign (Gula Darah, Detak Jantung, dll).
   - GET  /api/vitals : Mengambil riwayat vital sign pasien untuk grafik Chart.js.

6. Pengumuman (Announcements)
   - POST /api/announcements : Membuat pengumuman kesehatan baru (Khusus Dokter/Admin).
   - GET  /api/announcements : Mendapatkan semua pengumuman yang aktif.
